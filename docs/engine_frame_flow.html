핵심 해석

  • 이 코드에서 "한 프레임"은 wWinMain의 while 루프 한 번이다. 메시지 처리 후 RenderBegin, panel->Update, RenderEnd, Present, SwapIndex 순서로 돈다.
  • 게임/에디터 로직은 현재 JScenePanel::Update에 모여 있다. 입력으로 카메라 transform과 선택 오브젝트 visible 상태를 바꾸고, 그 결과가 JScene의 dirty/event 상태로 남는다.
  • JRenderServer::SyncScene가 CPU 렌더 준비의 중심이다. 씬을 JFrameSnapshot으로 만들고, RenderObject 이벤트를 바탕으로 JDrawItemCache를 갱신한 뒤, 카메라별 visible draw index 목록을 만든다.
  • JRenderer::Render는 FrameDesc를 GPU 리소스 DB에 동기화하고 현재 render path의 pass들을 실행한다. 에디터 패널 초기화에서 Deferred path로 바꾸므로 일반 실행은 GBuffer, Lighting, ForwardOverlay 순서다.
  • JCommandQueue는 프레임별 upload buffer와 descriptor heap을 reset하고, pass들이 기록한 command list를 닫아서 실행한 뒤 fence를 signal한다.
